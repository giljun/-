# Spring_DI

## 객체의존성

- 현재 객체가 다른 객체와 참조하고 있다면, 현재 객체는 다른 객체의 의존성을 가진다.
  - 두 객체 간의 긴밀한 결합이 생기고
  - 하나의 모듈이 변경되면, 다른 모듈까지 변경되어야한다.
  - Unit Test 작성이 어려워진다.



## 의존성 주입

- Framwork에 의해 객체의 의존성이 주입되는 패턴
  - 동적 주입으로 객체 간의 결합이 낮아진다.
  - Spring에서 지원하는 IoC의 형태
- 주입 방법 3가지
  - 생성자를 통한 전달
  - 메서드를 통한 전달
  - 멤버 변수를 통한 전달



## 장점

- 의존성 감소 
  - 유지보수 및 변경에 용이
- 재사용성 증가
  - 인터페이스 기반 설계
  - 코드 변경 필요 없음
- 더 많은 테스트 코드를 작성할 수 있다
- 코드의 가독성 증가
